# Поиск статей справки по вопросу пользователя

Решение задачи информационного поиска для RAG-пайплайна справочного центра. Для каждого вопроса пользователя скрипт возвращает ранжированный список из 10 `article_id`. Итоговый результат на платформе: **MAP@10 = 0.565010**.

## Структура данных

Скрипт ожидает каталог `candidate_data` с файлами:

- `articles.f` — статьи с колонками `article_id`, `title`, `body`;
- `calibration.f` — размеченные запросы `query_id`, `query_text`, `ground_truth`;
- `test.f` — тестовые запросы `query_id`, `query_text`.

Исходный набор не добавлен в репозиторий: его нужно скачать на странице задания.

## Подход

### Обработка HTML

HTML статьи преобразуется в обычный текст:

1. HTML-сущности декодируются через `html.unescape`.
2. Теги и ссылки удаляются регулярными выражениями.
3. Текст приводится к нижнему регистру, `ё` заменяется на `е`.
4. Удаляются служебные символы и повторяющиеся пробелы.
5. Заголовок статьи повторяется пять раз, чтобы он сильнее влиял на релевантность, чем длинное тело статьи.

### Признаки и ранжирование

Решение объединяет несколько локальных методов без внешних API и больших языковых моделей:

- символьный TF-IDF по `char_wb`-граммам длины 2–5 для устойчивости к опечаткам;
- словарный TF-IDF по униграммам и биграммам после русского Snowball-стемминга;
- kernel ridge-регрессия по размеченным вопросам; используется смесь символьного и словарного сходства, после чего ядро поэлементно возводится в квадрат;
- голосование 10 ближайших размеченных вопросов;
- независимый поиск по заголовкам и очищенному HTML статей с помощью символьного и словарного TF-IDF;
- объединение моделей через reciprocal-rank fusion;
- небольшие детерминированные поправки для однозначных формулировок: стоимость доставки, промокоды, упаковка, отмена заказа, ошибки оплаты и другие частые случаи.

На выходе выбираются 10 уникальных статей с наибольшим итоговым баллом.

## Проверка качества

Базовые параметры подбирались на `calibration.f`. Для оценки использовался leave-one-out: текущий размеченный вопрос исключался из обучающей части, после чего для него строился топ-10 и рассчитывался AP@10. Среднее значение по всем вопросам использовалось как локальный MAP@10.

Дополнительно проверялось:

- ровно 500 строк в итоговом CSV;
- точные колонки `query_id` и `answer`;
- соответствие и порядок всех `query_id` файлу `test.f`;
- ровно 10 идентификаторов без повторов в каждом ответе;
- существование каждого идентификатора в `articles.f`.

## Какие ошибки анализировались

- **Опечатки и разные словоформы.** Добавлены символьные n-граммы и русский стемминг.
- **Длинный и шумный HTML.** Теги удаляются, а заголовок получает больший вес.
- **Разговорные вопросы, не похожие на текст статьи.** Используются похожие размеченные обращения из `calibration.f`.
- **Несколько правильных статей для одного вопроса.** Модель обучается как многометочная, а в ответ всегда возвращается полный топ-10.
- **Редкие статьи без достаточного числа примеров.** Сохранён отдельный поиск непосредственно по корпусу статей.
- **Похожие намерения.** После анализа ошибок добавлены узкие правила, не заменяющие основную модель, а лишь немного корректирующие её баллы.

## Запуск

```bash
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -r requirements.txt
python solution.py --data-dir candidate_data --output answer.csv
```

После запуска рядом появится `answer.csv`. Скрипт автоматически проверит его формат и завершится с ошибкой, если обнаружит неправильные `article_id`, повторы или пропущенные запросы.

## Воспроизводимость

При одинаковых входных файлах алгоритм детерминирован и создаёт тот же `answer.csv`, который был отправлен на платформу. Случайные модели и внешние сетевые запросы не используются.
